# Somali style guide + glossary

**Status: DRAFT — needs native Somali shopkeeper review before it governs the
rewrite.** This is the contract the AI-assisted Somali rewrite follows so the app
reads naturally, stays consistent, and fits tight UI. It is seeded from the terms
the app already uses (so nothing here is novel jargon) plus deliberate revisions.

Companion artifacts:
- `l10n-review/key_roles.json` / `worklist.csv` — every key's role + length budget,
  regenerated by `python3 l10n-review/build_worklist.py`.
- `test/l10n/arb_parity_test.dart` — enforces en/so key parity (release blocker).

---

## 1. Principles

1. **Purpose over literal.** Translate what the label is *for*, not word-for-word.
   A button is a command; a confirm title is a question; a toast is a "done"
   report. The English is a reference, not a template.
2. **Plain shop words, no accounting jargon** (CLAUDE.md). Not "receivable/ledger/
   post" → "money the customer owes", "saved". The reader is a non-tech shopkeeper.
3. **Short wins.** Somali words run longer than English; tight UI (home tiles,
   buttons, chips) wraps or clips. Hit the budget in §3 — drop filler words, prefer
   one strong word over two.
4. **One word per concept** (§4). The app must not say "Iibin" in one place and
   "Iib" in another for the same idea.
5. **Native review is the release gate.** The AI produces a strong draft + the
   reason for each non-literal choice; a native speaker approves before it ships.

## 2. Register by role (the form each kind of label takes)

| Role | Form | Example (EN → SO) |
|---|---|---|
| Button | **imperative verb** | SAVE → KAYDI · VOID → TIRTIR · ADD LINE → KU DAR |
| Bottom SAVE | one verb | SAVE → KAYDI |
| Home tile | one noun/verb | Sale → Iibin · Expense → Qarashaad |
| App-bar / title | short **noun phrase** | Reports → Warbixino · Payment → Lacag |
| Confirm dialog title | **question** | Void this sale? → Iibkan tirtir? |
| Confirm dialog body | plain instruction sentence | "This can't be undone." → "Lama soo celin karo." |
| Chip / segment / badge | **very short noun** | Cash → Kaash · All → Dhammaan |
| Field label | noun phrase | Shop name → Magaca dukaanka |
| Hint / helper | short instruction | Search… → Raadi… |
| Toast (success) | **past-tense report** | Saved → Waa la kaydiyay |
| Toast (validation) | imperative/explanatory | "Add an item first" → "Marka hore alaab ku dar" |
| Metric / row label | noun, ≤ a few words | Sales total → Wadarta iibka |
| Empty / error | plain action-oriented sentence | (longest; can wrap) |

## 3. Length budgets (Somali, characters)

Tighter than English on tiles/chips because Somali is longer and these clip.

| Role | Budget | Lint? |
|---|---|---|
| Home tile | ≤ 14 (1 word ideal) | yes |
| Bottom SAVE | ≤ 12 | yes |
| Chip / segment / badge | ≤ 12 | yes |
| Button | ≤ 18 (≤ 2 words) | yes |
| App-bar / title | ≤ 20 | yes |
| Confirm dialog title | ≤ 30 | yes |
| Field label / metric | ≤ 24 | yes |
| Hint / helper / toast | flexible | no |
| Dialog body / empty / error | flexible (may wrap) | no |

When a faithful Somali word can't fit, in order: (a) a shorter synonym, (b) drop a
filler word, (c) accept a widely-understood English term (e.g. "Barcode", "App").
Flag any forced choice in the worklist Notes.

## 6. Process

1. `python3 l10n-review/build_worklist.py` → regenerate `worklist.csv`
   (Area, Key, Role, MaxChars, English, Somali-current, **Somali (AI)**, **Notes**).
2. AI fills **Somali (AI)** + **Notes** per this guide, batched by Area, within
   MaxChars; flags forced/uncertain choices.
3. **Native reviewer** edits in place + signs off per batch.
4. Apply approved values into `lib/l10n/app_so.arb` (values only — keys never
   change), `flutter gen-l10n`, `flutter analyze`, `flutter test`
   (parity must stay green), on-device check of the tightest strings.
5. The overflow report (script output) should trend to **0**; once it does, promote
   the budget lint to a hard test.
